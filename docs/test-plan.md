## Affected Surfaces (there's no URL routing; every surface is a view/screen)

- **First-run wizard** — splash, font install progress, Publisher Profile (10 fields), optional LibreOffice setup, Ready screen. Test: can a first-time user reach a blank Issue Board without asking for help?
- **Issue Board** — pages + placements view. Drag templates onto slots. Drag articles onto templates. Reorder pages. Section grouping.
- **Articles tab** — imported articles list. Drag from here to Issue Board. Per-article metadata.
- **Classifieds tab** — queue + forms for 12 types + CSV bulk import with preview.
- **Ads tab** — inventory by slot type. Upload with aspect-ratio + DPI validation. Drag to page.
- **Images tab** — photo library. Filter by usage. Crop + rotate.
- **Templates tab** — read-only preview browser.
- **History panel** — snapshot timeline. Preview + restore.
- **Settings / Publisher Profile** — edit stored publisher config + fonts + backups + diagnostics.
- **Pre-export check screen** — runs print-readiness checklist, shows warnings + errors, no PPTX generation.
- **Export screen** — generates PPTX + runs OOXML validation + "Done. Open in PowerPoint" message.
- **Cover editor** — masthead, hero, cover lines, metadata, optional cover strip/corner ad.

## Key Interactions to Verify

- **Drag template from palette to empty page slot** → template fills empty article, auto-fit filters show match (on Issue Board).
- **Drag article from Articles tab to page** → auto-fit suggests best template; operator can switch via popover.
- **Drag already-placed article to another page** → move-with-confirmation dialog (per 4B).
- **Drop new template on filled slot** → swap-with-confirmation (per 4A).
- **Reorder pages via drag** → persistence + snapshot.
- **Bulk drop 20 docx files** → review screen with 17 succeeded / 3 failed (per 4C).
- **Ambiguous article (auto-fit 2 templates within 15% margin)** → appears in "needs review" queue after bulk import.
- **Start CSV classifieds import** → preview grid with valid + invalid + duplicate rows highlighted.
- **Submit classified form quickly twice** → one entry created (debounce + dedup per 4F).
- **Close classifieds form mid-entry then reopen** → draft restored (per 4E).
- **Switch typography pairing mid-issue** → UI says "change applies to next issue or re-render".
- **Switch page size mid-issue (after placements)** → UI blocks with "copy to new A5 issue" prompt (per 4H).
- **Cover edit after export** → "this will change previously-exported file" warning (per 4I).
- **Restore a historical snapshot** → issue reverts to that state AND creates a new snapshot at HEAD.
- **Recovery prompt on startup after force-kill** → Restore / Discard / Open-Fresh options (per Accepted Scope #4).
- **Click "Check my issue"** → warnings + errors shown; LibreOffice validator runs; no PPTX generated.
- **Click "Export"** → PPTX generated atomically; success message with "open in PowerPoint" instruction.
- **Re-export an already-exported issue** → prompt: regenerate from current state or prior snapshot? (per 1H).
- **Font missing at editor open** → block with install prompt (per 1I).

## Edge Cases

- **Empty states**: zero articles in Library; zero classifieds queued; zero placed pages; zero snapshots (new issue).
- **High-count states**: 500-image Library; 2000 queued classifieds; 100-page issue.
- **Stale state**: drag article X from Library while another tab deleted it (stale cache). Expected: Library refreshes; drag gracefully fails with message.
- **Network absence**: the app is offline-first; no network should ever be attempted (verify no outbound requests in integration tests).
- **Disk full**: export mid-write, snapshot mid-write, image import mid-write → atomic rollback, clear message, no partial state.
- **Extremely long classified**: 3000-character public notice → triggers Extended Notice page (per 2C).
- **Extremely long title**: 200-character headline → doesn't break Pretext or PPTX render.
- **Unicode edge cases**: zero-width joiner, combining marks, mixed bidi, Arabic numerals in Devanagari body.
- **Mixed-script article**: Hindi body with English brand names → renders cleanly in both editor preview and PPTX.
- **CSV with embedded commas in quoted fields** → parsed correctly (papaparse handles).
- **CSV with UTF-16 encoding** → EncodingError with clear message.
- **Image at 149 DPI print size** → reject unless override (ResolutionBelowHardMin).
- **Image at 250 DPI** → warning but accept.
- **Ad with 3% aspect mismatch** → hard reject with expected-vs-got message.
- **Docx with zip-slip attempt** → rejected safely by mammoth; no file system escape.
- **SQLite corruption on next open** → restore-from-backup prompt (per 2B).
- **Corrupt blob detected during editor session** → operator dialog (per 1.4).
- **Electron force-killed mid-auto-save** → recovery prompt (per Accepted Scope #4); last clean snapshot is loadable.
- **LibreOffice missing at pre-export check** → setup prompt, Check-my-issue disabled until installed.
- **Font install fails due to Gatekeeper** → remediation step.
- **Operator substitutes Latin font that lacks Devanagari glyphs** → warning before applying substitution.
- **Two classifieds with identical (phone + billing ref)** → dedup flag in preview.
- **Zero-result auto-fit**: article too long/short/wrong-lang/wrong-image-count → clear explanation of why no templates matched.

## Critical Paths (must work every release)

1. **First issue end-to-end**: first-run wizard → Publisher Profile → new A4 issue → drop 3 docx → auto-fit + place → add 10 classifieds via form → add 2 ads → cover → Check-my-issue passes → Export → PPTX opens in PowerPoint → exports to PDF → visually correct. This is the golden path; every release must pass this manually or via a recorded Playwright E2E.
2. **Bulk issue**: drop 15 docx at once → bulk review → proceed → all auto-placed → export succeeds.
3. **CSV classifieds import**: drop 500-row CSV → preview → import valid → packed into Classifieds section → export succeeds.
4. **Crash recovery**: force-kill Electron mid-edit → relaunch → recovery prompt → Restore → state matches pre-crash.
5. **Devanagari Phase 3 gate**: Hindi article → editor preview matches PPTX render in Office 365 macOS visually.
6. **Re-export with prior snapshot**: export issue v1 → edit → export again → prompt → pick "regenerate from earlier snapshot" → matches v1.
7. **Pretext → PPTX mapping fidelity (Phase 2 gate)**: Standard Feature A4 English → prediction vs PowerPoint render identical per OV1 pixel-level framing.
